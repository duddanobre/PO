// Seção 03

Programação Linear

Otimização onde tudo é linear

O que é Programação Linear?

Z = 5x + 3y

Linear

Dobrou o investimento? O retorno dobra. Relação proporcional e previsível. Solução ótima sempre num vértice.

Z = x² + sen(y)

Não-linear

Dobrou o investimento? O retorno pode triplicar ou cair. Relação imprevisível. Solução ótima pode estar em qualquer ponto.

PL funciona quando: objetivo e restrições são proporcionais (lineares). Se dobrar a produção, dobra o custo e o lucro. Sem surpresas.

“O que o Seu João controla?”

Quantos bolos produzir?

x1 = quantidade de pães

“O que ele quer maximizar?”

Simplificado

Simplificado

Como a gente fala

Queremos o maior lucro possível

Max Z = 2x1 + 5x2

Seu João → Notação Formal

O mesmo problema, agora na linguagem que vocês vão encontrar nos livros e artigos.

De onde vem cada elemento

c = vetor de lucros (coeficientes do objetivo)

R$ 2 por pão, R$ 5 por bolo → c = [2, 5]

A = matriz de coeficientes das restrições

Farinha: 1 pão + 2 bolos | Tempo: 1 pão + 1 bolo

A = [[1, 2], [1, 1]]

b = vetor de limites (lado direito das restrições)

10kg de farinha, 6h de forno → b = [10, 6]

Forma compacta

Max Z = cTx
sujeito a:
Ax ≤ b
x ≥ 0
Mesma coisa, só que compacto. Com 100 variáveis, essa notação salva.

Temos o modelo. E agora?

Precisamos achar a solução ótima. Com 2 variáveis, dá pra resolver no gráfico.

Passo 5

Achar os pontos onde cada reta cruza os eixos

Passo 6

Achar onde as retas se cruzam (sistema)

Passo 7

Testar quais pontos são viáveis

Passo 8

Calcular Z em cada vértice → o maior ganha

Vamos lá: peguem a restrição de farinha. Se x1 = 0, quanto dá x2?

Passo 4 → Pontos nos Eixos

Cada restrição vira uma reta no gráfico. Pra desenhar, achamos onde ela cruza os eixos.

Restrição 1: Farinha → x1 + 2x2 ≤ 10

Se x1 = 0 (zero pães):

0 + 2x2 = 10 → x2 = 5 → ponto (0, 5)

Se x2 = 0 (zero bolos):

x1 + 0 = 10 → x1 = 10 → ponto (10, 0)

Restrição 2: Tempo → x1 + x2 ≤ 6

Se x1 = 0:

0 + x2 = 6 → x2 = 6 → ponto (0, 6)

Se x2 = 0:

x1 + 0 = 6 → x1 = 6 → ponto (6, 0)

Lógica: zeramos uma variável por vez pra achar onde a reta toca cada eixo. Dois pontos → uma reta.

Passo 4 → Interseção das Retas

Onde as duas retas se cruzam? Resolvemos o sistema:

x1 + 2x2 = 10
x1 + x2 = 6
  1. Subtraímos a 2ª da 1ª → x2 = 4
  2. Substituímos na 2ª → x1 + 4 = 6 → x1 = 2
  3. Ponto de interseção → (2, 4)
Agora temos todos os vértices candidatos: (0, 0), (0, 5), (2, 4), (6, 0). Qual dá mais lucro?

Passo 5 → Testar Viabilidade

Ponto Farinha (≤10) Tempo (≤6) Viável?
(0, 0) 0 ✓ 0 ✓ Sim
(0, 5) 10 ✓ 5 ✓ Sim
(2, 4) 10 ✓ 6 ✓ Sim
(6, 0) 6 ✓ 6 ✓ Sim
(10, 0) 10 ✓ 10 ✗ Não
(0, 6) 12 ✗ 6 ✓ Não

Passo 6 → Calcular Z nos Vértices

Vértice x1 x2 Z = 2x1 + 5x2
A 0 0 R$ 0
B 0 5 R$ 25 ⭐
C 2 4 R$ 24
D 6 0 R$ 12
Resposta: 0 pães e 5 bolos → Lucro máximo = R$ 25

// Seção 05

Método Simplex

Solução algébrica → funciona com qualquer número de variáveis

Por que Simplex?

  • Gráfico só funciona com 2 variáveis. Problemas reais têm centenas.
  • Simplex faz a mesma coisa, pula de vértice em vértice, algebricamente.
  • George Dantzig, 1947. Um dos algoritmos mais importantes da história.

Simplex: qual o plano?

Vamos resolver o mesmo problema do Seu João, agora com Simplex.

0. Forma padrão

Transformar ≤ em = com variáveis de folga

1. Tableau inicial

Montar a tabela com todas as variáveis

2. Quem entra?

Mais negativo na linha Z

3. Quem sai?

Teste da razão: menor b ÷ coluna

4. Pivotear

Operações de linha pra zerar a coluna

5. É ótimo?

Tem negativo na linha Z? Se sim, repete

Passo 0

O que fazer? → Forma Padrão

O Simplex não trabalha com ≤. Precisa de =

Como? Adicionar uma variável de folga em cada restrição.
Folga = quanto do recurso sobra.

Exemplo:   x1 + 2x2 ≤ 10

Vira:   x1 + 2x2 + x3 = 10

Façam no caderno: coloquem as 2 restrições do Seu João na forma padrão.

Passo 0 → Solução

Forma Padrão

x1 + 2x2 + x3 = 10    (x3 = folga farinha)
x1 + x2 + x4 = 6    (x4 = folga tempo)
Se x1=0, x2=0 → x3=10 (sobra toda farinha), x4=6 (sobra todo tempo)

Passo 1

O que fazer? → Montar o Tableau

Pegar os coeficientes de cada equação e jogar numa tabela.

Cabeçalho: Base | x1 | x2 | x3 | x4 | b
Base inicial: as variáveis de folga (x3, x4)
Linha Z: coeficientes com sinal trocado (−2, −5)
Montem o tableau no caderno. Uma linha por equação + linha Z embaixo.

Passo 1 → Solução

Tableau Inicial

Base x1 x2 x3 x4 b
x3 1 2 1 0 10
x4 1 1 0 1 6
Z −2 −5 0 0 0
  • Básicas: x3=10, x4=6 → Não básicas: x1=0, x2=0 → Z=0
  • Traduzindo: não produz nada, sobra tudo, lucro zero.

Passo 2

O que fazer? → Quem ENTRA?

Olhem a linha Z do tableau que vocês montaram.

Regra: valor negativo na linha Z = dá pra melhorar o lucro.
O mais negativo é o que mais melhora → ele entra.
Qual variável entra? x1 (−2) ou x2 (−5)?

Passo 2 → Solução

Quem ENTRA na base?

  • Linha Z: −2 e −5
  • Mais negativo = −5 (coluna x2)
  • x2 entra na base! (produzir bolo é mais lucrativo)
Coluna pivô = x2

Passo 3

O que fazer? → Quem SAI?

Teste da razão mínima: divide b pelo valor na coluna pivô.

Regra: quem tem a menor razão (b ÷ coluna pivô) sai.
Só valores positivos entram no teste.
Façam: 10 ÷ ? e 6 ÷ ? → Qual é menor?

Passo 3 → Solução

Quem SAI da base?

Base x2 b Razão
x3 2 10 10/2 = 5 ← menor
x4 1 6 6/1 = 6
  • Menor razão = 5 → x3 sai!
  • Pivô = 2 (cruzamento coluna x2 × linha x3)

Passo 4

O que fazer? → Pivotear

coluna pivô ↓
Base x1 x2 x3 x4 b
x3 1 2 1 0 10 ← linha pivô
x4 1 1 0 1 6
Z −2 −5 0 0 0

Objetivo: fazer a coluna x2 ficar com 1 na linha pivô e 0 nas outras.

Primeiro

Dividir a linha pivô pelo pivô (2)

Depois

Zerar x2 nas outras linhas

Façam as 3 operações no caderno. Pivô = 2.

Passo 4 → Solução

Pivotear

Por que zerar? Cada variável na base precisa aparecer em uma única linha. Zeramos a coluna pra isolar x2 na sua linha → assim lemos o valor direto no b.
Fórmula:   Nova linha = Linha antiga − (valor na coluna pivô) × Nova linha pivô
  • 4a Dividir linha pivô por 2:
    [1, 2, 1, 0, 10] ÷ 2 = [½, 1, ½, 0, 5]
  • 4b Zerar x2 na linha x4 (valor = 1):
    [1, 1, 0, 1, 6] − 1×[½, 1, ½, 0, 5] = [½, 0, −½, 1, 1]
  • 4c Zerar x2 na linha Z (valor = −5):
    [−2, −5, 0, 0, 0] − (−5)×[½, 1, ½, 0, 5] = [½, 0, 5/2, 0, 25]

Passo 5

O que fazer? → É ótimo?

Montem o novo tableau com os resultados do pivoteamento.

Regra: olhem a linha Z do novo tableau.
Tem algum valor negativo? Se não → ótimo encontrado!
Montem o novo tableau e me digam: a solução é ótima?

Passo 5 → Solução

É ótimo?

Base x1 x2 x3 x4 b
x2 ½ 1 ½ 0 5
x4 ½ 0 −½ 1 1
Z +½ 0 +5/2 0 25
Nenhum valor negativo na linha Z → não dá pra melhorar!
x1=0, x2=5, Z = R$ 25. Mesmo resultado do gráfico!

Simplex : Resumo

  1. Forma padrão → adicionar variáveis de folga
  2. Tableau inicial → montar a tabela
  3. Quem entra? → mais negativo na linha Z
  4. Quem sai? → teste da razão (menor b ÷ coluna)
  5. Pivotear → zerar a coluna com operações de linha
  6. Repetir → até não ter negativo na linha Z

x1 = LpVariable("paes", lowBound=0)

prob += x1 + 2*x2 <= 10
prob += x1 +   x2 <= 6

Cada médico tem uma nota de preferência de 1 a 10 por turno.

Objetivo: montar a escala que maximize a satisfação total dos médicos.

Quantas variáveis? Que tipo? É PL ou PI? Modelem e resolvam com PuLP.
